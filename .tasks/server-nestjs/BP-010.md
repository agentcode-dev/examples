# BP-010 — `AGENTCODE_PRISMA_CLIENT` token not exported from public module surface

**Type:** missing export
**Severity:** low — consumer workarounds exist, but DI is clunky
**Affects:** `@agentcode-dev/agentcode-nestjs@0.1.0`

## Problem

The library declares an injection token:

```ts
// src/constants/tokens.ts
export const AGENTCODE_PRISMA_CLIENT = Symbol('AGENTCODE_PRISMA_CLIENT');
```

Used internally to wire `prismaClient` through `AgentCodeModule.forRoot`.

But `AgentCodeModule.coreExports()` does NOT include this token. Consumer code like:

```ts
@Injectable()
export class TenantRewriteMiddleware {
  constructor(
    @Inject(AGENTCODE_PRISMA_CLIENT) private readonly prisma: PrismaClient,
  ) {}
}
```

...fails with:

```
Nest can't resolve dependencies of the TenantRewriteMiddleware (?). Please
make sure that the argument Symbol(AGENTCODE_PRISMA_CLIENT) at index [0]
is available in the AppModule context.
```

## Root cause

`src/agentcode.module.ts`:

```ts
private static coreExports() {
  return [
    AgentCodeConfigService,
    PrismaService,
    ResourceService,
    // ...
    // AGENTCODE_PRISMA_CLIENT  ← missing
  ];
}
```

The provider IS registered (inside `build()` it's added with `{ provide: AGENTCODE_PRISMA_CLIENT, useValue: ... }`) but not exported, so downstream modules can't inject it.

## Workaround

Use `PrismaService` instead — it IS exported. Inject `PrismaService` and call `.client` to get the underlying PrismaClient instance:

```ts
constructor(private readonly prismaService: PrismaService) {}

private get prisma(): PrismaClient {
  return this.prismaService.client as PrismaClient;
}
```

Tolerable, but:
- Consumers need to know the `.client` escape hatch exists.
- Lose the typed `PrismaClient` (the `PrismaService` generic parameter is needed → subclass or cast).

## Proposed fix

Add `AGENTCODE_PRISMA_CLIENT` to `coreExports()`:

```ts
private static coreExports() {
  return [
    AGENTCODE_PRISMA_CLIENT,
    AgentCodeConfigService,
    PrismaService,
    // ...
  ];
}
```

Also export the constant from the barrel:

```ts
// src/index.ts
export { AGENTCODE_PRISMA_CLIENT, AGENTCODE_CONFIG, AGENTCODE_MODULE_OPTIONS } from './constants/tokens';
```

(Already exported from `src/index.ts` — the problem is only the module `exports` array.)
